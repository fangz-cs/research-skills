# Synthesis Mode (领域综述模式)

主流程在 `SKILL.md`。本文件只在用户想把**读过的多篇论文织成一个领域的整体认知**时读取——触发于"把我读的这些串起来 / 这个领域现在什么格局 / 综述一下 / build a field map / compare what I've read"。

目标:让用户得到的是**对一个领域的整体判断**,而不是一堆互不相连的阅读报告。

## 核心原则:笔记是源,全景是派生

**单篇笔记(`*-notes.md` + 其 YAML front-matter)是唯一真相源;领域全景 `_landscape.md` 是派生产物。**

全景**重新生成**,不要反复就地改写——就地 LLM 改写会像传话游戏一样越改越烂、积 cruft。源保持干净,理解演进了就重生成一版全景。`_index.md` 那张 append-only 账本(格式见 `SKILL.md` 的 Workflow 第 7 步)是全景的廉价输入。

## 生成全景的步骤

1. **圈定范围**:领域全景默认针对一个 `field`。读取该范围下**所有** `*-notes.md` 的 front-matter(需要时连正文一起)+ `_index.md`。用户若指定子主题,按 `topics` 标签过滤后再综述(分类是视图,在这里动态发生,不依赖目录结构)。front-matter 是聚合的主要依据——`topics`/`method_family` 用来聚类,`relates_to` 用来连线,`confidence` 用来定权重。
2. 若已存在旧的 `_landscape.md`,先读它,以便结尾给出"本次新增 / 变化了什么"。
3. 按下面结构**重新生成**全景(不是在旧文件上缝补)。**保持这个 skill 的批判性**:不是罗列论文,而是给出有判断、能找到跨论文异味的领域地图。
4. 交付方式同 `SKILL.md` 的 Output Conventions → Delivery(可下载 / 可预览,或无文件系统时直接输出在对话里)。

## 全景结构(`_landscape.md`)

```markdown
# <领域> 全景 (截至 <date>,基于 N 篇笔记)

## 一句话格局
<这个领域当前的整体态势,2–3 句>

## Approach 分类与谱系
<把方法按 method_family 聚类;用 relates_to 画出谁延续谁,主要有哪几条线>

## 核心问题与状态
<这个领域在攻什么问题;哪些已基本解决,哪些仍开放>

## 争议与矛盾(最有价值)
<哪些论文的结论 / 主张彼此打架(来自 relates_to 的 contradicts);谁更可信,为什么>

## 共识
<已被多篇印证、可当作前提的结论>

## 跨论文异味
<整个领域层面的问题:如"全员在已饱和的 benchmark X 上刷分""整条线建立在没人验证的假设 Y 上""baseline 互相挑软柿子">

## 空白与机会
<聚合各篇的 future work,识别"没人在做但重要"的方向;区分小改 vs 真研究机会>

## 我对这个领域的判断
<逐渐成形的个人 thesis:最值得下注的方向、最被高估的方向>

## 本次更新
<相对上一版全景,新增了哪些论文、出现了哪些新争议、哪些开放问题被关闭>
```

## 规模与诚实

- 笔记很多时(如 >20 篇),可按用户给定的子主题先筛 `_index.md` 再综述,不必硬塞全部。
- front-matter 缺失或 `confidence: low` 的笔记,在全景里要**降权**,并在相应判断旁标注"基于信息不足的笔记"。
- "诚实 > 完整"在综述层同样适用——**不要把零散笔记夸大成一个比实际更确定的领域结论**。N 篇覆盖不全时,明说全景是片面的。
